# VTX JSON files

Pre-computed vertex array metadata extracted from reference O2Rs.
Used by `zapd_to_torch.py --vtx-json` to add VTX entries to YAML files
without requiring a reference O2R at YAML generation time.

One JSON per asset configuration (matches YAML directory names):

| File | XML Dir | Status |
|------|---------|--------|
| `pal_gc.json` | GC_NMQ_PAL_F | Done |
| `pal_mq.json` | GC_MQ_PAL_F | Needs O2R |
| `ntsc_u_gc.json` | GC_NMQ_NTSC_U | Needs O2R |
| `ntsc_u_mq.json` | GC_MQ_NTSC_U | Needs O2R |
| `ntsc_j_gc.json` | GC_NMQ_NTSC_J | Needs O2R |
| `ntsc_j_gc_collection.json` | GC_NMQ_NTSC_J_CE | Needs O2R |
| `ntsc_j_mq.json` | GC_MQ_NTSC_J | Needs O2R |
| `pal_gc_dbg.json` | GC_NMQ_D | Needs O2R |
| `pal_mq_dbg.json` | GC_MQ_D | Needs O2R |
| `ntsc_u_1-0.json` | N64_NTSC_10 | Needs O2R |
| `ntsc_u_1-1.json` | N64_NTSC_11 | Needs O2R |
| `ntsc_u_1-2.json` | N64_NTSC_12 | Needs O2R |
| `pal_1-0.json` | N64_PAL_10 | Needs O2R |
| `pal_1-1.json` | N64_PAL_11 | Needs O2R |

## Generating a VTX JSON

```sh
python3 soh/tools/extract_vtx.py <reference.o2r> soh/vtx/<name>.json
```

The reference O2R must be generated by OTRExporter (Shipwright build b48e5f7)
for the corresponding ROM version.
