# VTX JSON files

Pre-computed vertex array metadata extracted from reference O2Rs.
Used by `zapd_to_torch.py --vtx-json` to add VTX entries to YAML files
without requiring a reference O2R at YAML generation time.

One JSON per unique asset configuration. NTSC J/U share the same data
within each version (verified by hash comparison).

| File | Entries | Covers |
|------|---------|--------|
| `pal_gc.json` | 7011 | PAL GC non-MQ |
| `pal_mq.json` | 7012 | PAL GC MQ |
| `pal_1-0.json` | 7011 | PAL N64 1.0 |
| `pal_1-1.json` | 7011 | PAL N64 1.1 |
| `pal_gc_dbg.json` | 7068 | PAL GC Debug |
| `pal_mq_dbg.json` | 7070 | PAL MQ Debug |
| `ntsc_gc.json` | 7019 | NTSC J+U GC + J Collection |
| `ntsc_mq.json` | 7019 | NTSC J+U MQ |
| `ntsc_1-0.json` | 7019 | NTSC J+U N64 1.0 |
| `ntsc_1-1.json` | 7019 | NTSC J+U N64 1.1 |
| `ntsc_1-2.json` | 7019 | NTSC J+U N64 1.2 |

## Generating a VTX JSON

```sh
python3 soh/tools/extract_vtx.py <reference.o2r> soh/vtx/<name>.json
```

Reference O2Rs generated by OTRExporter (Shipwright build b48e5f7).
